Backend Temel Kavramları

Request–Response Cycle

Client → Server → DB → Server → Client akışını bil.

Routing

URL → hangi handler çalışacak; REST/GraphQL entry noktası.

Middleware

Auth, logging, rate limiting, error handler, session, body parser.

Controller / Handler

İsteği alır, iş mantığını çalıştırır, yanıtı hazırlar.

ORM / Query Builder

Tabloyu koda map eder. Örn: Prisma, Entity Framework, Django ORM, Eloquent, Hibernate, TypeORM.

API Türleri

REST, GraphQL, gRPC, WebSocket → protokol ve veri modeli kararları.

I/O-bound vs CPU-bound

I/O dış kaynağı bekler; CPU hesap yapar. Dil seçimi ve ölçek için kritik ayrım.

Concurrency

Birden fazla işin ilerlemesi; event loop, goroutine, thread pool ile yönetilir.

Parallelism

Gerçekten aynı anda çekirdeklerde çalışma; CPU-bound işleri hızlandırır.

Framework Mantığı

Routing + middleware + controller + config + DI birleşimi.

Monolith vs Microservice

Monolith = tek app; Microservice = küçük bağımsız servisler.

Scaling

Vertical: CPU/RAM artır. Horizontal: yeni instance çoğalt.

Caching

Redis, Memcached ile sıcak veriyi hızlandır.

Environment & Config

.env, secrets yönetimi, ortam bazlı config.
